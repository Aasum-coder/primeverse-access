### Inbound Email Auto-Verification

## Key Learnings & Bugs Found

- **Next.js 307 on POST** — webhook senders don't follow redirects on POST. `skipTrailingSlashRedirect: true` is the fix.
- **Resend inbound payload structure** — email ID is at `data.email_id` (not `data.id` or `payload.id`). Attachments have full metadata (`id`, `filename`, `content_type`) in the webhook BUT the file content is NOT inline — must fetch via `resend.emails.receiving.attachments.get({ emailId, id })` to get a signed `download_url`.
- **React Flow cannot be overlay-blocked** — even with a 100%-cover overlay on top, React Flow's internal handlers still receive input. The fix is conditional render (don't mount WorkflowCanvas at all for non-admins).
- **`email_sends` schema mismatch** — table has `user_id → distributors.id` (not `distributor_id`) and no `lead_id` column. Per-lead email counts depend on an embedded Supabase join that only works if the FK is defined in the schema.
- **`.profile-grid`** — `align-items: start` is critical when columns have very different heights, otherwise the shorter column stretches.
- **Attachment ID comes from the webhook payload directly** — no need to call `list()` first; the attachment's `id` inside `data.attachments[]` is the same ID required by the `get()` endpoint.
